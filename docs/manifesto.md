# ReXamples Manifesto

ReXamples is a native ReAction teaching project for AmigaOS 3.2+.

It exists to show how ReAction should be learned: directly, clearly, and in plain C89.

ReXamples does not imitate other GUI systems.  
It teaches native AmigaOS GUI construction on its own terms.

Each example is a focused lesson.  
Each lesson teaches one concept well instead of many concepts poorly.

Every lesson must be small enough to study, clear enough to trust, and disciplined
enough to reuse as a reference.

Resource ownership is always explicit.  
Initialization is deliberate.  
Cleanup is mandatory.  
Failure paths are part of the design.

ReXamples treats event-driven structure as fundamental, not optional.  
A GUI program without a clear lifecycle is not a finished lesson.

Abstraction is allowed only when it improves clarity.  
Nothing important should be hidden behind convenience code.

Lessons must teach correct native habits, not shortcuts.  
They are not demos of what merely works, but references for what should be done.

ReXamples values readability over cleverness, structure over improvisation,
and system integration over toolkit imitation.

Its purpose is not comparison, emulation, or adaptation.  
Its purpose is to define a clear native baseline for disciplined ReAction
development on AmigaOS 3.2+.
